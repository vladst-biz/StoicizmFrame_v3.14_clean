# Протокол создания файлов в StoicizmFrame v3.14_clean

## 1. Обсуждение
- Зачем создаётся файл (назначение, роль в архитектуре).
- Где создаётся (папка, узел, версия).

## 2. Создание
- Файл создаётся только после согласования.
- Сразу фиксируется название и место.

## 3. Проверка кодировки
- Файл проверяется визуально.
- Кодировка: UTF-8 LF (без BOM).

## 4. Фиксация
- Добавление в Git:
  git add <имя_файла>
  git commit -m "<TAG>: <описание назначения>"
- Тегирование узла при необходимости:
  git tag <версия-узел>
  git push origin master
  git push origin <версия-узел>

## 5. Документирование
- В коммите и описании указывается назначение файла.
- При необходимости создаётся релиз на GitHub.
