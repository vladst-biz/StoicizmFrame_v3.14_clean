# ЗАКОН STOICIZMFRAME №1  
## О РЕЖИМЕ WORKSPACE И СТАБИЛЬНОСТИ PYTHON-ИНФРАСТРУКТУРЫ

### Статья 1. Открытие проекта
Проект StoicizmFrame открывается исключительно через файл workspace:
StoicizmFrame.code-workspace
Открытие проекта как «папки» запрещено.

### Статья 2. Единая точка истины для настроек
Все настройки Python, среды, терминала и анализа кода хранятся только в workspace-файле.

### Статья 3. Автоматическая активация среды
Каждый новый терминал обязан автоматически активировать виртуальную среду .venv311.
Отсутствие активации считается нарушением закона.

### Статья 4. PYTHONPATH и видимость проекта
Путь D:/StoicizmFrame_v3.14_clean должен быть доступен интерпретатору Python.
Импорты внутри проекта обязаны работать без относительных хаосов.

### Статья 5. Единый интерпретатор
Проект использует только один интерпретатор:
D:\StoicizmFrame_v3.14_clean\.venv311\Scripts\python.exe

### Статья 6. Терминал как часть архитектуры
Каждый новый терминал должен начинаться с:
(.venv311) PS D:\StoicizmFrame_v3.14_clean>

### Статья 7. Git-чистота
После каждого архитектурного узла выполняется:
git add .
git commit -m "Architectural node: workspace stabilization"

### Статья 8. Непрерывность наследия
Каждое изменение инфраструктуры фиксируется в dev_log.txt и HERITAGE.md.

### Статья 9. Неизменность закона
Закон является фундаментом инфраструктуры StoicizmFrame.
Любые изменения допускаются только через новый архитектурный узел.
