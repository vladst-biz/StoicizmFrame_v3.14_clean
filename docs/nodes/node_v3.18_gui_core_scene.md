# Node v3.18 — GUI/Core/SCENE Stabilization

## ENTRY

Узел v3.18 фиксирует момент, когда StoicizmFrame переходит от разрозненных модулей к цельной архитектуре.
GUI-контур, Core-контур и SCENE-оркестрация синхронизированы и образуют единый жизненный цикл.

## WORK

1. Стабилизирован GUI-контур:
   - TopBar: состояние, версия, цветовая индикация статуса пайплайна
   - LeftPanel: выбор направления
   - CenterPanel: управление пайплайном, режимы, состояния running/idle
   - RightPanel: мониторинг статуса, QC, Health, лог, прогресс
   - StatusBar: UX-ритм ENTRY → WORK → LEGACY, env, project, direction, mode, pipeline
   - main_window: дирижёр жизненного цикла GUI

2. Стабилизирован Core-контур:
   - нормализованные статусы
   - структурированный лог с timestamp
   - обработка ошибок
   - хранение параметров запуска
   - единый жизненный цикл start → execute → done/error

3. Введена SCENE-архитектура:
   - BaseScene — базовый контракт сцены
   - SCENE_001 (ENTRY) — подготовка данных и контекста
   - SCENE_002 (WORK) — основной рабочий этап (будущий мультимодальный модуль)
   - SCENE_003 (LEGACY) — финализация, упаковка, подготовка результатов
   - оркестрация сцен в PipelineCore: ENTRY → WORK → LEGACY

## LEGACY

Узел v3.18 создаёт фундамент для мультимодального пайплайна StoicizmFrame:
- SCENE_002 становится точкой расширения для Foundry, RAG, Voice, LayoutComposer
- GUI полностью синхронизирован с Core и Adapter
- жизненный цикл пайплайна стал предсказуемым и управляемым
- лог стал структурированным и пригодным для анализа и отладки
- проект готов к переходу в мультимодальный режим (v3.19+)
