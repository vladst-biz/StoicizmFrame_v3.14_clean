# ИНФРАСТРУКТУРНЫЙ ПРОТОКОЛ STOICIZMFRAME
## Стандарты кода, структуры и запуска

### 1. Единый стиль кода
Проект StoicizmFrame использует единый стиль форматирования:
- Black — форматирование
- Ruff — линтинг
- isort — сортировка импортов

Все файлы Python должны соответствовать этим правилам.

### 2. Структура каталогов
Структура проекта является фиксированной и включает:
- src/ — ядро и базовые модули
- services/ — сервисные компоненты
- scenarios/ — сценарные блоки
- scenes/ — сценовые элементы
- pipeline/ — обработка и генерация
- adapters/ — интеграции
- interface/ — взаимодействие
- infrastructure/ — низкоуровневые механизмы

Изменение структуры допускается только через архитектурный узел.

### 3. Протокол запуска
Запуск проекта осуществляется через единый entrypoint:
python src/main.py

Все вспомогательные скрипты должны быть оформлены как модули.

### 4. Протокол импортов
Импорты внутри проекта должны быть абсолютными:
from infrastructure.azure_voice_service import AzureVoiceService

Относительные импорты запрещены.

### 5. Единый pipeline
Pipeline генерации контента должен быть оформлен как модуль:
pipeline/content_pipeline.py

Все этапы (donors → scenario → voice → layout → publish) должны быть стандартизированы.

### 6. Единый формат конфигурации
Конфигурация проекта хранится в:
config.json
.env

Изменение формата конфигурации допускается только через архитектурный узел.

### 7. Единый протокол тестирования
Тесты оформляются в каталоге:
tests/

Каждый модуль должен иметь тестовый файл.

### 8. Единый протокол логирования
Логирование оформляется через модуль:
core/logger.py

Формат логов фиксирован.

### 9. Неизменность протокола
Инфраструктурный протокол является обязательным для всех будущих модулей.
Изменения допускаются только через новый архитектурный узел.
