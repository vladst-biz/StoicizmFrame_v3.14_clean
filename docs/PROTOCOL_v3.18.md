# PROTOCOL v3.18 — CORE Integration

## Назначение
Установить правила взаимодействия между модулями CORE, определить формат данных, порядок вызовов и границы ответственности.

## Структура CORE
- engine — сборка контента
- visual — визуальные блоки и композиции
- voice — генерация и обработка голоса
- publisher — публикация в YouTube и Telegram
- analytics — сбор и анализ метрик

## Общие правила
- Все модули работают в UTF-8 LF
- Каждый модуль имеет свой MODULE_PASSPORT.md
- Взаимодействие происходит через стандартизированные структуры данных (JSON)
- Ошибки передаются в виде исключений
- Логи обязательны

## Базовый порядок вызовов
SCENE → engine → visual → voice → publisher → analytics

## Расширение
В v3.21 будет добавлен модуль reframer (между SCENE и engine) для уникализации и адаптации формулировок без изменения смыслов.
