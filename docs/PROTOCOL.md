# PROTOCOL — Правила StoicizmFrame

## Создание файлов
- Новые узлы создаются только через оркестратор.
- Имена фиксируются в формате SCENE_001.md, REPORT_vX.Y.md и т.д.
- Все ключи и конфиги хранятся в .env.

## Правки
- Изменения допускаются только в рабочей папке.
- Архивные версии не редактируются.

## Кодировка
- Все файлы сохраняются в UTF‑8 без BOM.
- Конец строки фиксирован как LF.
- Проверка на «крякозябры» обязательна.
- Если файл случайно создан с BOM или CRLF — он удаляется и пересоздаётся заново.

## Запреты
- Нет дублирования скриптов.
- Нет хаотичных правок без фиксации в LEGACY.md.
