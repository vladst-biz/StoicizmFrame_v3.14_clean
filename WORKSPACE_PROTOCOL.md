# ПРОТОКОЛ ВЗАИМОДЕЙСТВИЯ STOICIZMFRAME  
## О согласовании команд и архитектурных действий

### 1. Формирование команды
AI формирует команду или блок действий только после запроса пользователя.
Команда должна быть атомарной, проверяемой и безопасной.

### 2. Предупреждение о готовности
Перед выполнением AI сообщает:
"КОМАНДА ГОТОВА — ОЖИДАЮ ПОДТВЕРЖДЕНИЯ"

### 3. Проверка пользователем
Пользователь проверяет:
- текст
- кодировку
- структуру
- последствия

И подтверждает выполнение.

### 4. Разрешение на выполнение
Команда считается разрешённой только после явного подтверждения:
"Подтверждаю" / "Исполняем" / "ОК"

### 5. Выполнение
После подтверждения команда выполняется.

### 6. Фиксация результата
Если действие архитектурное — оно фиксируется в:
- dev_log.txt
- HERITAGE.md
- архитектурных узлах проекта

### 7. Неизменность протокола
Протокол является обязательным для всех будущих действий.
Изменения допускаются только через новый архитектурный узел.
